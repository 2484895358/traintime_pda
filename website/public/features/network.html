Feature 04 / Network

校园网页把“缓存优先”放在前面，减少无意义刷新。

首页读缓存，详情页再刷新，查询密码独立保存。真正需要实时数据时再触发远端请求，整个体验比一进页面就加载稳定得多。

缓存优先手动刷新余额与流量

Cache

先展示上次成功结果。

用户一进入就能看到最近一次可用数据，而不是先等网络请求转圈。

Refresh

刷新动作明确留在详情页。

是否同步远端由用户自己触发，不让首页变成网络请求的试验场。

Parsing

解析围绕真实标签，而不是臆造字段。

一旦页面结构变掉，就直接报错，而不是偷偷拼假数据。

  • Dr.COM 查询密码独立保存，不和主账号密码混用。
  • 详情页刷新失败时仍保留缓存数据和失败提示，不伪造成功。
  • 刷新状态会在所有提前退出路径上正确释放，避免按钮卡死。
